## 📌 Overview

Welcome to my **Software Security Lab** repository – a collection of hands-on experiments conducted during the 6th semester of my B.Tech in Computer Science & Engineering (Cybersecurity). This lab focuses on understanding, exploiting, and defending against common software vulnerabilities. Each experiment includes step-by-step outputs, observations, and analysis.

> 🧪 **Current contents**: PDF outputs of Experiments 01–05.

## 🧪 Experiments

| # | Experiment Title | Objective | Output File |
|---|----------------|-----------|--------------|
| 01 | **Buffer Overflow Basics** | Understand stack-based buffer overflow, overwrite return addresses, and execute shellcode. | [Experiment 01 Outputs.pdf](./Experiment%2001%20Outputs.pdf) |
| 02 | **Format String Vulnerabilities** | Exploit printf() family bugs to read/write arbitrary memory. | [Experiment 02 Outputs.pdf](./Experiment%2002%20Outputs.pdf) |
| 03 | **Privilege Escalation via SUID Binaries** | Identify and exploit misconfigured SUID binaries on Linux. | [Experiment 03 Outputs.pdf](./Experiment%2003%20Outputs.pdf) |
| 04 | **Static Analysis & Fuzzing** | Use tools like `gcc` sanitizers, `clang-static-analyzer`, and basic fuzzing to find bugs. | [Experiment 04 (Outputs).pdf](./Experiment%2004%20(Outputs).pdf) |
| 05 | **Secure Coding & Cryptographic APIs** | Implement input validation, avoid dangerous functions, and use libsodium/OpenSSL correctly. | [Experiment 05 (Outputs).pdf](./Experiment%2005%20(Outputs).pdf) |
