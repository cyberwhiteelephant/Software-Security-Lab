## 📌 Overview

This repository contains my **hands-on lab work** for the *Software Security Lab* course during the 6th semester of my B.Tech in Computer Science & Engineering (Cybersecurity).  

The lab covers **10 core experiments** – from basic code analysis and input-based attacks to threat modeling, incident response, and using real security tools (Wireshark, OWASP ZAP, OpenSSL, etc.).

## 📚 Complete Lab Syllabus (10 Experiments)

| # | Experiment Title | Key Activities & Tools |
|---|----------------|------------------------|
| 1 | **Static Code Analysis for Security Flaws** | Analyze C/Python snippets for buffer overflow, hardcoded creds, unsafe input handling. Use manual review + static analyzers. |
| 2 | **Input‑Based Attacks & Defenses** | Perform basic SQLi & XSS on test apps (DVWA/local). Implement input validation & sanitization. |
| 3 | **Cryptographic Hashing & Secure Password Storage** | Compare MD5, SHA‑256; implement hashing + salting in Python/OpenSSL. |
| 4 | **Secure Code Review – Login System** | Review a login module for plain‑text passwords, missing input checks. Suggest fixes. |
| 5 | **Network Traffic Inspection** | Capture HTTP login transmissions with Wireshark/tcpdump. Identify plain‑text vulnerabilities. |
| 6 | **Threat Modeling (STRIDE)** | Model a web app (e.g., online bookstore). Identify threats, attack surfaces. Draw diagrams (free tools or by hand). |
| 7 | **Authentication with Password Hashing** | Build a simple login form with hashed password verification (no plain‑text storage). |
| 8 | **Vulnerability Scanning (OWASP ZAP)** | Passive scan a web app for outdated software, default creds, misconfigurations. |
| 9 | **Browser Artifacts & Privacy Risks** | Inspect history, cookies, saved data via DevTools / Browser History Viewer. Understand user tracking. |
| 10 | **Incident Response from Logs** | Analyze sample access logs for brute‑force attempts. Draft an incident report. |
